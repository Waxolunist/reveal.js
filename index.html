  • In Python geschrieben
  • Läuft auf Docker
  • 1-bit bezieht sich nicht auf die Dateigrösse, wär ein bisschen wenig, sondern auf den Farbraum.
  • Bitthresholding - Er machts halt SW
  • Thumbnails sind nichts spezielles, aber der ist besonders effizient.

Rückblick

  • Vaadin als Framework schwierig im App Bereich
    • Server Dependency
    • Daher Offline sehr schwierig zu implementieren
    • Vaadin Version auf Client und Server müssen übereinstimmen
  • Probleme mit Altlasten bei z.B. Qualität, Softwareversionen
  • Scrum nur bedingt anwendbar mit gleichzeitigem Betrieb

Hybrid-Ansatz kann man auch in Frage stellen. Ich persönlich würde wieder einen Hybrid-Ansatz wählen, jedoch nicht mehr Vaadin als Framework. Aber das ist Gegenstand einer offenen Diskussion, welche wir in den nächsten Wochen zusammen mit Daniel führen werden.
